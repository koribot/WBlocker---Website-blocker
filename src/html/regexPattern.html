Regex Pattern

The regular expression pattern which is matched against the network request url. The pattern is anchored at both ends.

If omitted, all urls are matched. An empty string is not allowed.

Note: Only one of Url Pattern or Regex Pattern can be specified.

Here are some examples of regular expression patterns that can be used with Regex Pattern:

  • .*example\.com.*: Matches any URL that contains "example.com".
  • ^https://.*\.example\.com/.*: Matches any URL that starts with "https://" and ends with ".example.com".
  • ^https://(www\.)?example\.com/.*: Matches any URL that starts with "https://" and ends with ".example.com", with an optional "www." subdomain.
  • ^https://.*\.example\.com/path/.*: Matches any URL that starts with "https://" and contains ".example.com/path/" in the path.

Note: The Regex Pattern must be composed of only ASCII characters. This is matched against a url where the host is encoded in the punycode format (in case of internationalized domains) and any other non-ascii characters are url encoded in utf-8.
